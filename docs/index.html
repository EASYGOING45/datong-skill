需求模糊，不知道从哪开始

方向不清晰，无从下手，越想越乱

推荐 知难行易

核心思路：先行动获取反馈，再修正认知。不要等完美方案，想清楚"最难的那一步是什么"就够了。

"知之非艰，行之惟艰，此古人之大谬也。" —— 孙中山
⚖️

两个方案纠结，不知道选哪个

各有道理，陷入分析瘫痪

推荐 三民框架

核心思路：从三个维度审视——谁所有（民族）、谁治理（民权）、谁受益（民生）。三维一贯，缺一不可。

"民有、民治、民享。" —— 孙中山
🔱

多人协作，权责不清

决策、执行、审查混在一起，谁都说是别人的问题

推荐 五权制衡

核心思路：五种职能（执行/规划/验证/选拔/监察）应当分离，避免既当运动员又当裁判员。

"在外国的规模之上更加一层中国的特色。" —— 孙中山
🏛️

系统设计，架构边界摇摆

不知道哪些该自己做，哪些该放出去

推荐 权能区分

核心思路：明确区分决策权（属于用户/资方）和执行力（属于执行者/AI）。既不越权，也不推责。

"用人民的四个政权，来管理政府的五个治权。" —— 孙中山
📣

要推动一个变革，遇到阻力

方案是好的，但别人不买账，不知道怎么推进

推荐 唤起民众

核心思路：孤军不可能成功。要做两件事——唤起相关方理解并愿意参与；找到"以平等待我"的力量建立联盟。

"必须唤起民众及联合世界上以平等待我之民族，共同奋斗。" —— 孙中山
⚡

大项目压得喘不过气，不知道怎么拆解

问题太多太杂，不知道从哪切入

推荐 毕其功于一役

核心思路：多个维度互相关联时，不要枝节修补，集中资源一次性系统解决，撕开突破口。

"要把工业革命与社会革命，毕其功于一役。" —— 孙中山
📊

事情太多，分不清主次节奏

资源有限，不知道先做哪个，怎么排优先级

推荐 三阶段推进

核心思路：任何深刻变革经历三阶段——破旧立新期 → 教育培养期 → 制度自治期。急于跳过中间阶段，制度就无法扎根。

"军政→训政→宪政" —— 孙中山
🔥

遇到重大挫折，想放弃

方案被否、反复失败、信心受挫

推荐 愈挫愈奋

核心思路：失败不是终点，是认识深化的过程。每次失败后总结经验、调整方法、重新出发。

"革命尚未成功，同志仍须努力。" —— 孙中山 · 总理遗嘱（1925）
🌱

要帮助别人成长，但不知道怎么做

想赋能团队或他人，但不知道从哪入手

推荐 济弱扶倾

核心思路：不是直接给答案，而是引导他自己站起来。强者有责任帮助弱者，但帮的方式是让对方变强。

"聪明才智越大者，当服千万人之务，造千万人之福。" —— 孙中山
🔄

中西方方案打架，不知道怎么融合

外国方法好，但不知道能不能本地化

推荐 融贯中西

核心思路：取各家之长融为己用，但融合的前提是深入理解双方，而非浮光掠影或简单二选一。

"固有的东西，如果是好的，当然是要保存，不好的才可以放弃。" —— 孙中山

行动先于完美认知。真正的困难不在于"做"，而在于"知道该做什么"。遇到不确定时，先行动获取信息，不等完美方案。

核心原则

  • 真正的"难"是认知上的难，不是行动上的难
  • 先做起来，在行动中获取真实反馈
  • 决策前先区分"我知道我不知道"和"我不知道我不知道"

适用场景

分析瘫痪、犹豫不决、畏难不前、需求模糊、技术选型迷茫

不适用场景

高风险不可逆决策（手术、重大财务决策）、高度规范化流程

原文引用

"知之非艰，行之惟艰，此古人之大谬也。" —— 孙中山《建国方略》（1917）
"吾心信其可行，则移山填海之难，终有成功之日。" —— 孙中山

配套工具

→ 用「三阶段推进」设计行动节奏
→ 用「五权制衡」建立验证机制

不盲从外来方法，也不固守已有方式。取各家之长融为己用，但融合的前提是深入理解双方，而非浮光掠影。

核心原则

  • 先理解对方为什么这么做，再谈融合
  • 融合是再创造，不是简单叠加
  • 区分"形式"和"实质"，学实质而非形式

适用场景

技术选型、架构设计、方案权衡、流程改进、外包/采购决策

不适用场景

需要快速标准化执行、没有足够时间深入研究理解的场景

原文引用

"固有的东西，如果是好的，当然是要保存，不好的才可以放弃。" —— 孙中山《三民主义》（1924）
"我们要学外国，是要迎头赶上，不要向后跟着。" —— 孙中山

配套工具

→ 用「三民框架」评估融合方案的利弊
→ 用「知难行易」避免过度研究空谈

任何系统性方案可从三个维度审查——谁所有、谁治理、谁受益。三维一贯，缺一不可。

核心原则

  • 民族（所有权）：谁拥有这个系统/方案的最终决策权？
  • 民权（治理权）：谁参与决策过程？机制是否公正？
  • 民生（受益权）：谁从这个方案中获益？成本如何分配？

适用场景

系统设计评审、需求分析、架构决策、API 边界设计、数据权限设计

不适用场景

简单的一次性执行任务、个人独立决策场景

原文引用

"民有、民治、民享。" —— 孙中山（借鉴林肯语）

配套工具

→ 用「五权制衡」设计治理结构
→ 用「权能区分」明确执行边界

五种职能应当分离独立运行：执行、规划、验证、选拔、监察。建立系统性检查与平衡机制。

核心原则

  • 执行者不能审查自己的工作
  • 规划权与执行权应分离（避免"计划赶不上变化"）
  • 验证结果必须被真正使用才能产生价值

适用场景

代码审查流程、质量保障体系、流程设计、权限管理、大型多人协作项目

不适用场景

快速原型验证、小团队高信任环境、个人项目

原文引用

"在外国的规模之上更加一层中国的特色。" —— 孙中山（指五权分立）

配套工具

→ 用「三民框架」评估权力结构的合理性
→ 用「知难行易」避免过度设计流程

明确区分决策权（属于用户/资方）和执行力（属于执行者/AI）。既不越权替用户做决定，也不推责在自己的能力范围内充分发挥。

核心原则

  • AI 的职责是执行，不是决策
  • 当 AI 发现决策权归属不清晰时，应当指出并请求明确
  • 执行者有义务在权限范围内做到最好，但不应擅自超越

适用场景

人机协作边界设计、API 设计、权限系统设计、团队角色分工

不适用场景

用户明确授权 AI 全权代理的场景、高度紧急无法及时反馈的场景

原文引用

"用人民的四个政权，来管理政府的五个治权。" —— 孙中山

配套工具

→ 用「三民框架」明确权力来源
→ 用「五权制衡」建立执行监督机制

孤军奋战不可能成功。需要做两件事：唤起相关方理解问题并愿意参与；找到所有"以平等待我"的力量，建立统一战线。

核心原则

  • 方案再好，没人理解就没法落地
  • "唤起"不是说服，是让对方看到自己的利益所在
  • 统一战线需要识别真正的盟友和潜在的反对者

适用场景

跨团队协作推进、方案推广、开源运营、技术选型投票、招聘与文化建设

不适用场景

高度机密或时间极度紧迫的任务、需要保密的决策

原文引用

"必须唤起民众及联合世界上以平等待我之民族，共同奋斗。" —— 孙中山《总理遗嘱》（1925）

配套工具

→ 用「三民框架」识别各方利益
→ 用「三阶段推进」设计推进节奏

当多个维度的问题互相关联时，不要枝节片断地修补，而要一次性系统解决。避免走西方"先革命再改革"的弯路。

核心原则

  • 识别问题的"根"而不是"枝"——找到那个解决了其他都会跟着解决的问题
  • 集中资源单点突破，而不是分散兵力四处修补
  • 需要判断"这是系统性问题还是偶发问题"

适用场景

大规模重构、技术债务清理、系统迁移、遗留系统现代化、组织架构调整

不适用场景

资源极度有限需要渐进演进、风险不可控的高敏感系统

原文引用

"要把工业革命与社会革命，毕其功于一役。" —— 孙中山

配套工具

→ 用「知难行易」判断最难的那一步
→ 用「三阶段推进」设计执行节奏

任何深刻变革不可能一步到位。经历三阶段：破旧立新期 → 教育培养期 → 制度自治期。急于跳过中间阶段，制度就无法扎根。

核心原则

  • 第一阶段：破旧立新，集中力量解决最紧迫问题
  • 第二阶段：教育培养，参与者逐渐掌握新规则
  • 第三阶段：制度自治，形成自我运转的机制

适用场景

新技术规范推行、团队转型、大型迁移、并购整合、制度建设

不适用场景

已经成熟的常规运营、紧急危机响应

原文引用

"军政→训政→宪政" —— 孙中山《建国大纲》（1924）

配套工具

→ 用「知难行易」确定每阶段的突破口
→ 用「五权制衡」设计阶段性验收机制

失败不是终点，是认识深化的过程。每次失败后不是放弃，而是总结经验、调整方法、重新出发。

核心原则

  • 区分"方法失败"和"方向失败"——前者调整方法，后者需要重新思考方向
  • 每次挫折后必须形成可操作的下一步，而不是简单重复
  • 韧性来自对失败模式的识别，不是来自意志力

适用场景

方案被否后的重新设计、多次迭代不通过的困境、信心重建

不适用场景

不可逆的重大失败、资源已经耗尽的情况

原文引用

"既不可以失败而灰心，亦不能以困难而缩步。" —— 孙中山
"革命尚未成功，同志仍须努力。" —— 孙中山《总理遗嘱》（1925）

配套工具

→ 用「知难行易」分析失败的真实原因
→ 用「三民框架」重新审视方案基础假设

强者有责任帮助弱者。系统的公平性、对弱势用户的关怀、对边缘场景的覆盖，比单纯追求效率和性能更重要。为最弱势使用者设计。

核心原则

  • "帮助"不是替代，是赋能——让对方自己站起来
  • 在效率和公平之间，优先保障基本可达性
  • 边缘场景往往揭示系统最深层的缺陷

适用场景

可访问性设计（a11y）、国际化/本地化、向下兼容、技术普惠产品设计

不适用场景

性能敏感的实时系统、对安全性要求极高的基础设施

原文引用

使用指南

Clone 之后，怎么真正用起来？

四种使用方式，总有一种适合你

🤖

方式一：对话中直接调用

在对话中直接说明场景，让 AI 自动调用相关方法论

你说：
"我要做一个积分体系，团队两个人吵得很凶，各有道理，该怎么分析？"
AI 自动用「三民框架」+「知难行易」帮你分析
Step 1：确保 datong-skill 已加载到 AI 的上下文中
Step 2：直接描述你的问题场景
Step 3：AI 会识别适用的方法论并应用
📋

方式二：按需查询单个方法论

遇到具体问题时，针对性查询某个方法论

你说：
"用五权制衡分析一下我们团队的代码审查流程"
AI 加载五权制衡方法论，针对性分析
推荐：查看上方「按场景找方法论」区块，找到最匹配的方法论
⚡

方式三：入口 Skill 路由

通过「天下为公」入口 Skill，让 AI 自动判断使用哪个工具

将 skills/tianxia-weigong/SKILL.md 内容作为 AI 的 system prompt 或 pre-issue prompt
入口 Skill 会：
  • 根据你的问题描述判断场景
  • 自动选择最合适的方法论组合
  • 提示你使用方法的最佳时机
🔧

方式四：作为工作流组件

在复杂场景中组合多个方法论，形成完整工作流

1. 唤起民众 → 识别利益相关方
2. 三民框架 → 三维评估方案
3. 五权制衡 → 设计执行机制
4. 愈挫愈奋 → 遇阻时调整策略
提示：组合使用时请注意方法论的适用边界，不是越多越好
快速安装

一条命令，开始使用

→ 详细5分钟上手指南

Bash 
git clone https://github.com/EASYGOING45/datong-skill
cd datong-skill
claude --plugin-dir .

持久化加载（加入 ~/.zshrc 或 ~/.bashrc）：

alias claude-datong='claude --plugin-dir /path/to/datong-skill'

# 使用时：
claude-datong
# 然后正常对话，AI 会自动加载 datong-skill 方法论

OpenClaw 有两种加载方式：

方式 A：作为 System Skill（推荐）

配置 
# 在 OpenClaw 配置中添加 skills 路径：
{
  "skills": {
    "load": {
      "extraDirs": ["/path/to/datong-skill/skills"]
    }
  }
}

然后告诉 AI："我现在加载了 datong-skill，遇到了 XXX 问题"

方式 B：作为 System Prompt

Terminal 
# 将入口 Skill 内容加入 system prompt：
cat skills/tianxia-weigong/SKILL.md

复制输出的内容，粘贴到 OpenClaw 的 system prompt 配置中

方式 C：ClawHub（v0.2.0 即将支持）

# 即将支持一行命令安装：
openclaw skill install datong-skill
