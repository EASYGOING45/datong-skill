01认识论

知难行易

Know Hard, Act Easy

行动先于完美认知。真正的困难不在于"做"，而在于"知道该做什么"。遇到不确定时，先行动获取信息，不等完美方案。

"知之非艰，行之惟艰，此古人之大谬也。"
适用：分析瘫痪、犹豫不决、畏难不前

02认识论

融贯中西

East-West Synthesis

不盲从外来方法，也不固守已有方式。取各家之长融为己用，但融合的前提是深入理解双方，而非浮光掠影。

"固有的东西，如果是好的，当然是要保存，不好的才可以放弃。"
适用：技术选型、架构设计、方案权衡

03治理方法

三民框架

Three Principles Analysis

任何系统性方案可从三个维度审查——谁的（所有权）、谁治理（决策结构）、谁受益（价值分配）。三维一贯，缺一不可。

"民有、民治、民享。"
适用：系统设计评审、需求分析、架构决策
查看详情 →
04治理方法

五权制衡

Five Power Checks

执行、规划、验证、选拔、监察——五种职能应当分离独立运行，建立系统性检查与平衡机制。

"在外国的规模之上更加一层中国的特色。"
适用：代码审查、质量保障、流程设计
查看详情 →
05治理方法

权能区分

Power-Capability Split

明确区分决策权（属于用户）和执行力（属于执行者）。既不越权替用户做决定，也不推责在自己的能力范围内充分发挥。

"用人民的四个政权，来管理政府的五个治权。"
适用：角色分工、权限设计、人机协作边界

06革命战略

唤起民众

Awaken and Unite

孤军奋战不可能成功。需要做两件事：唤起相关方理解问题并愿意参与；找到所有"以平等待我"的力量，建立统一战线。

"必须唤起民众及联合世界上以平等待我之民族，共同奋斗。"
适用：跨团队协作、方案推广、开源运营
查看详情 →
07革命战略

毕其功于一役

Single-Stroke Revolution

当多个维度的问题互相关联时，不要枝节片断地修补，而要一次性系统解决。避免走西方"先革命再改革"的弯路。

"要把工业革命与社会革命，毕其功于一役。"
适用：重构、迁移、技术债务清理
查看详情 →
08革命战略

三阶段推进

Three Phase Governance

任何深刻变革不可能一步到位。经历三阶段：破旧立新期 → 教育培养期 → 制度自治期。急于跳过中间阶段，制度就无法扎根。

"军政→训政→宪政"
适用：新技术规范推行、团队转型、大型迁移
查看详情 →
09精神纪律

愈挫愈奋

Rise from Defeat

失败不是终点，是认识深化的过程。每次失败后不是放弃，而是总结经验、调整方法、重新出发。"既不可以失败而灰心，亦不能以困难而缩步。"

"革命尚未成功，同志仍须努力。"
适用：方案被否、多次失败、信心受挫
查看详情 →
10精神纪律

济弱扶倾

Aid the Weak

强者有责任帮助弱者。系统的公平性、对弱势用户的关怀、对边缘场景的覆盖，比单纯追求效率和性能更重要。为最弱势使用者设计。

"聪明才智越大者，当服千万人之务，造千万人之福。"
适用：可访问性、向下兼容、技术普惠

快速安装

一条命令，开始使用

支持 Claude Code / Cursor / OpenClaw，开箱即用

Bash 
git clone https://github.com/EASYGOING45/datong-skill
cd datong-skill
claude --plugin-dir .

设置别名持久加载：

# 加入 ~/.zshrc 或 ~/.bashrc
alias claude='claude --plugin-dir /path/to/datong-skill'
JSON 
"skills": {
  "load": {
    "extraDirs": ["/path/to/datong-skill/skills"]
  }
}

或在 ClawHub 中搜索 datong-skill 一键安装

Bash 
git clone https://github.com/EASYGOING45/datong-skill
