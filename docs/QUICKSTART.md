# 5 分钟上手大同 Skill

> 想快速上手？这份指南带你 5 分钟理解价值 + 开始使用。

---

## Step 1：安装（1 分钟）

**Claude Code 用户：**

```bash
git clone https://github.com/EASYGOING45/datong-skill
cd datong-skill
claude --plugin-dir .
```

**OpenClaw 用户：**

将 `skills/tianxia-weigong/SKILL.md` 的内容加入你的 system prompt，或通过 ClawHub 安装（v0.2.0 支持）。

安装完成后说一声"我要解决 XXX 问题"，AI 就会自动调用相关方法论。

---

## Step 2：你能获得什么（2 分钟）

遇到问题时，**先判断场景类型，再选择方法论**。

### 场景 → 方法论 速查表

| 你遇到的情况 | 推荐方法论 | 为什么用它 |
|---|---|---|
| **需求模糊，不知道从哪开始** | 「知难行易」 | 知道"难"在哪，行动就清晰了 |
| **两个方案纠结，不知道选哪个** | 「三民框架」 | 从民族、民权、民生三个维度评估 |
| **多人协作，权责不清** | 「五权制衡」 | 决策、执行、审查分离，不卡不乱 |
| **系统设计，架构摇摆不定** | 「权能区分」 | 把"能做"和"该做"分开，边界就清楚了 |
| **要推动一个变革，遇到阻力** | 「唤起民众」 | 让参与者看到收益，共识自然形成 |
| **大项目压得喘不过气，不知道怎么拆解** | 「毕其功于一役」 | 集中资源打透一个点，撕开突破口 |
| **事情太多，分不清主次** | 「三阶段推进」 | 先试点、再展开、后收尾，节奏清晰 |
| **遇到重大挫折，想放弃** | 「愈挫愈奋」 | 不是鸡汤，是失败模式分析和韧性建设 |
| **要帮助别人成长，但不知道怎么做** | 「济弱扶倾」 | 不是直接给答案，是引导他自己站起来 |
| **中西方方案打架，不知道怎么融合** | 「融贯中西」 | 找到最大公约数，而不是二选一 |

> 💡 **小贴士**：遇到复杂问题，可以组合使用。比如「唤起民众」+「三阶段推进」一起用。

## Step 3：第一次使用（2 分钟）

### 完整示例

**你说：**

> "我要做一个社区产品，但团队里两个人对要不要做积分体系吵得很凶，我觉得各有道理，该怎么分析？"

**AI 用「三民框架」帮你分析：**

```
好的，我们用三民框架来拆解这个问题：

【民族视角】
- 积分体系是否符合产品长期定位？
- 用户群体是否吃这套？

【民权视角】
- 两个人的分歧本质是什么？
  → 是「积分有没有用」（认知问题）还是「我方案被否了不服」（情绪问题）？
- 各方是否有平等的表达机会？

【民生视角】
- 积分体系的成本收益比如何？
- 能否做一个最小实验先验证，再全量铺开？

【建议下一步】
用「知难行易」先明确"积分体系难在哪"，再用「三阶段推进」设计试点方案。
```

**你说：**

> "好，那就先试点，试点怎么设计？"

**AI 用「三阶段推进」继续：

```
试点设计：
1. 试点范围：选 5% 用户，观察 2 周
2. 核心指标：DAU 变化、用户主动打开积分页次数
3. 决策点：如果指标提升 > 10%，全量；否则重新诊断

具体 prompt 可以继续问我"用三阶段推进设计积分试点"。
```
