# 三民框架 · 决策树

## 什么时候用？

### ✅ 直接用

- 需要做系统设计评审、架构决策
- 需求分析时发现利益相关方不清晰
- 方案讨论中各方诉求冲突，需要仲裁框架
- 技术方案涉及所有权、治理权、价值分配三方
- 审查一个方案是否"公平"——谁拥有？谁决策？谁受益？

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 发现多个方案各有优劣不知选哪个 | → 配合 **five-power-checks** 做方案选拔 |
| 弱势群体需求被忽视 | → 配合 **aid-the-weak** 做弱势审视 |
| 决策后需要分阶段推进 | → 配合 **three-phase-governance** 做阶段设计 |
| 方案涉及多方博弈，力量不对等 | → 配合 **awaken-and-unite** 平衡利益 |

### ❌ 不适用

- **问题域非常小，只涉及单一角色**——不需要三方框架，简化更高效
- **纯技术实现问题，没有治理/所有权维度**——直接用技术方案就好
- **用户只需要执行，不需要理解利益结构**——避免过度分析

---

## 组合使用示例

```
用户问："这个系统设计评审，团队里两个人有分歧"
  → 三民框架 ✅（从所有权/决策权/受益权拆解）
  → 识别"谁拥有系统"→"谁有决策权"→"谁受益"
  → 发现分歧本质是治理权分配问题
  → 配合 five-power-checks 找最优分配方案
```
