# 愈挫愈奋 · 决策树

## 什么时候用？

### ✅ 直接用

- 方案被否决，需要重新站起来继续推进
- 多次尝试都失败，团队士气低落
- 项目遇到重大挫折（技术障碍、团队变动、外部依赖失败）
- 用户表达"不行就算了""换个方向吧"的放弃念头
- agent 自己遇到执行困难，需要精神支撑继续尝试

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 失败后需要重新分析问题 | → 配合 **three-principles-analysis** 做深度复盘 |
| 需要重新设计方案但不确定方向 | → 配合 **east-west-synthesis** 探索新方向 |
| 失败是因为资源/能力不足 | → 配合 **power-capability-split** 重新分配权能 |
| 失败后需要唤起团队重新凝聚 | → 配合 **awaken-and-unite** 重建团队信心 |

### ❌ 不适用

- **失败是因为方法根本错了**——不应该坚持，应该转向
- **用户明确决定放弃**——尊重用户决策，不强行推进
- **失败带来严重不可逆后果**——先处理后果，不要在情绪中继续行动
- **问题本身不可行**——承认失败，用 **three-principles-analysis** 重新评估

---

## 组合使用示例

```
用户说："这个方案又被否决了，我觉得是不是方向错了，想放弃"
  → 愈挫愈奋 ✅（转化失败为认识深化的机会）
  → 不放弃，但也不盲目重复
  → 用 three-principles-analysis 复盘：方案被否决的真正原因
  → 调整方向后继续，用 rise-from-defeat 保持行动力
```
