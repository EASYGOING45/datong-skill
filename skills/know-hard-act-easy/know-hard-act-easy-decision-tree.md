# 知难行易 · 决策树

## 什么时候用？

### ✅ 直接用

- 用户说"不知道从哪里开始"、"完全没思路"、"还没想清楚先不做"
- 用户陷入过度分析，反复讨论方案但不下结论
- 新项目冷启动，需要快速产出第一版而非要先研讨完备
- 任务被描述为"太难了""太复杂了""还没准备好"

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 知道要做但不确定方向对不对 | → 配合 **east-west-synthesis** 做方向探索 |
| 行动后遇到阻力或批评 | → 配合 **rise-from-defeat** 做失败转化 |
| 有多个可行方案不知选哪个 | → 配合 **five-power-checks** 做方案选拔 |
| 需要组织多人协作 | → 配合 **awaken-and-unite** 统一共识 |

### ❌ 不适用

- **任务本身确实存在未解决的技术依赖**——先解决外部依赖，否则行动只是假装在干活
- **需要精确安全验证的操作**（删除数据、公有API修改等）——先有明确边界再做
- **用户明确要求先有完整计划再执行**——尊重用户节奏，不强行推行动

---

## 组合使用示例

```
用户问："这个需求我想了很久，感觉好复杂不知道怎么做"
  → 知难行易 ✅（打破畏难，先动起来）
  → 在行动中逐步发现"难"在哪里
  → 配合 east-west-synthesis 拆解问题维度
  → 配合 power-capability-split 分离能做和不能做
```
