# 权能区分 · 决策树

## 什么时候用？

### ✅ 直接用

- 角色分工不清晰，"谁来决定"和"谁来执行"混淆
- agent 擅自做了应该由用户决策的事
- 用户抱怨"你为什么不先问我"或"你凭什么替我决定"
- 权限/权限设计需要明确决策者和执行者边界
- 任务出现推诿——双方都觉得对方应该做

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 决策者（用户）缺乏必要信息做判断 | → 配合 **know-hard-act-easy** 提供充分信息 |
| 执行者（agent）能力不足，需要先增强 | → 配合 **rise-from-defeat** 克服执行障碍 |
| 多人决策，权力分散，需要平衡 | → 配合 **three-principles-analysis** 分配权益 |
| 决策后需要推动执行，遇到阻力 | → 配合 **awaken-and-unite** 建立执行联盟 |

### ❌ 不适用

- **任务本身是纯执行，不需要决策**——直接执行就好，不需要讨论权力分工
- **紧急情况，来不及请示**——此时默认 agent 执行，事后汇报
- **用户明确表示"你决定就好"**——用户已主动转移决策权

## 组合使用示例

```
用户抱怨："这个需求你们没问我就自己定了"
  → 权能区分 ✅（明确：这是决策权，用户保留）
  → 厘清：哪些是用户该决定的，哪些是 agent 该执行的
  → 重新划分边界，达成新的权责协议
  → 配合 five-power-checks 验证新边界是否合理
```
