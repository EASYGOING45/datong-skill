# 唤起民众 · 决策树

## 什么时候用？

### ✅ 直接用

- 跨团队协作项目，利益相关方众多，协调困难
- 技术方案推广受阻，相关方不配合
- 开源项目运营，需要吸引贡献者
- 用户/干系人对方案价值理解不足，阻力来自认知而非技术
- 项目涉及多个团队，各自为战，无法形成合力

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 需要分阶段唤起，逐步加深参与 | → 配合 **three-phase-governance** 渐进式唤起 |
| 对方处于弱势，需要扶持 | → 配合 **aid-the-weak** 保护弱势方 |
| 唤起后遇到失败和阻力 | → 配合 **rise-from-defeat** 持续前进 |
| 唤起需要方向和原则指引 | → 配合 **three-principles-analysis** 建立共识基础 |

### ❌ 不适用

- **孤军奋战可以完成，不需要多方参与**——强行拉人反而降低效率
- **对方完全不感兴趣，无法唤起**——评估是否值得争取，考虑成本
- **紧急情况，无法等待达成共识**——先用 **single-stroke-revolution** 快速行动
- **团队已高度共识，只缺执行**——直接执行，不需要再唤起

---

## 组合使用示例

```
用户说："这个技术规范推行不下去，团队其他人不配合"
  → 唤起民众 ✅（统一共识，建立联盟）
  → 识别关键阻力方，理解对方为什么不配合
  → 找到共同利益点，构建价值叙事
  → 配合 three-phase-governance 分阶段达成全面共识
```
