# 大同工作流

将多个方法论 skill 串联为端到端的完整流程。

**红线提示：** 所有工作流仅用于 AI 工作场景的方法论指导，不涉及任何政治立场。

---

## 工作流一：系统设计评审流

**适用场景：** 新系统设计方案评审、重大架构变更评审

**步骤：**

1. **三民框架分析**（three-principles-analysis）
   - 输入：系统设计方案
   - 审查：谁的系统？谁治理？谁受益？三者是否对齐？
   - 输出：利益相关方分析报告

2. **融贯中西评估**（east-west-synthesis）
   - 输入：技术选型和架构决策
   - 审查：是否充分理解了各候选方案？融合是否有深度？
   - 输出：选型评估报告

3. **五权制衡检查**（five-power-checks）
   - 输入：系统流程设计
   - 审查：执行、规划、验证、选拔、监察是否分离？
   - 输出：质量保障检查报告（可交由 `five-power-checks/quality-review-prompt.md` subagent 执行）

4. **权能区分检查**（power-capability-split）
   - 输入：角色与权限设计
   - 审查：决策权和执行力是否正确划分？
   - 输出：角色权限审查报告

5. **济弱扶倾审查**（aid-the-weak）
   - 输入：用户体验设计
   - 审查：最弱势使用者是否被照顾到？
   - 输出：普惠性审查报告

**终止条件：** 所有审查通过，或发现问题并产生改进方案。

---

## 工作流二：变革推进流

**适用场景：** 推行新技术规范、团队转型、大型迁移

**步骤：**

1. **唤起民众**（awaken-and-unite）
   - 任务：识别利益相关方，唤起认知，建立联盟
   - 输出：支持者清单 + 沟通计划

2. **毕其功于一役 vs 三阶段推进**（评估决策）
   - 如果问题高度关联、资源充足 → 使用 single-stroke-revolution，制定一次性解决方案
   - 如果规模太大或风险不可控 → 使用 three-phase-governance，制定三阶段推进计划
   - 输出：推进策略（一步到位 or 分阶段）

3. **知难行易**（know-hard-act-easy）
   - 任务：不等完美方案，先在小范围行动获取反馈
   - 输出：快速原型 / 试点结果

4. **五权制衡**（five-power-checks）
   - 任务：为变革过程建立独立的验证和监察机制
   - 输出：变革质量保障计划

5. **愈挫愈奋**（rise-from-defeat）—— 按需激活
   - 触发条件：推进过程中遇到重大阻碍
   - 任务：结构化复盘 → 提取新认识 → 调整方案 → 重新出发

**终止条件：** 变革目标达成，或进入 three-phase-governance 第三阶段（制度自治）。

---

## 工作流三：危机应对流

**适用场景：** 方案被全面否决、项目严重受阻、团队信心崩塌

**步骤：**

1. **愈挫愈奋**（rise-from-defeat）
   - 任务：承认失败 → 分离情绪 → 结构化复盘 → 提取新认识
   - 输出：复盘报告 + 新的认识

2. **知难行易**（know-hard-act-easy）
   - 任务：基于新认识，找到成本最低的下一步行动
   - 输出：最小行动计划

3. **唤起民众**（awaken-and-unite）
   - 任务：重新评估支持力量，争取新的盟友
   - 输出：新的支持力量地图

4. **融贯中西**（east-west-synthesis）
   - 任务：引入之前没考虑过的新视角和新方法
   - 输出：新方案或改进方案

**终止条件：** 走出危机并形成新的可执行计划，或明确判断方向需要调整。
