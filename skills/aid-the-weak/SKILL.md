# 济弱扶倾

> "中国如果强盛起来，我们不但是要恢复民族的地位，还要对于世界负一个大责任。"
> —— 孙中山《三民主义·民族主义》

> "聪明才智越大者，当服千万人之务，造千万人之福。"
> —— 孙中山

**红线提示：** 本方法仅用于软件工程、系统设计等 AI 工作场景的方法论指导，不涉及任何政治立场。

## 核心要义

强者有责任帮助弱者。能力越大，责任越大。

天下为公，不是空谈——它意味着你做的每一个决定，都要考虑那些"全无能力者"是否也能受益。系统的公平性、对弱势用户的关怀、对边缘场景的覆盖，都比单纯追求效率和性能更重要。

孙中山将社会成员分为三类：先知先觉者、后知后觉者、不知不觉者。先知先觉者的责任不是俯视后两者，而是帮助他们。

## 操作步骤

1. **识别"最弱势使用者"** —— 在你的系统/方案中，谁是能力最弱、资源最少、最容易被忽略的使用者？
   - 技术能力最弱的用户
   - 设备/网络条件最差的用户
   - 最不了解系统规则的新用户
   - 有特殊障碍的用户（视觉、听觉、认知等）
2. **审视当前方案** —— 从最弱势使用者的视角重新审视：
   - 他们能使用这个功能吗？
   - 出错时他们能理解错误信息吗？
   - 他们有足够的信息做出正确决策吗？
3. **补齐短板** —— 针对发现的问题，制定改进措施：
   - 提供合理的默认值
   - 增加引导和提示
   - 优化降级方案和离线体验
   - 支持辅助技术（屏幕阅读器等）
4. **测试边缘场景** —— 用最弱势使用者的条件测试：低端设备、弱网环境、最小屏幕、辅助模式
5. **建立长期机制** —— 将"济弱扶倾"纳入评审清单，而非一次性检查

## 适用场景

- 可访问性（Accessibility）设计与审查
- 向下兼容策略：需要支持旧版本浏览器/设备
- API 设计：错误信息对调用方是否友好
- 技术普惠：降低使用门槛让更多人受益
- 文档与提示信息：是否对非专业用户友好
- 国际化与本地化

## 不适用场景

- 纯内部工具且用户群体能力一致时——但仍建议保持"考虑新人"的意识
- 面向专业开发者的底层库——此时更重要的是 API 的正确性和一致性
- 为弱势使用者做的适配会严重损害核心体验时——需要权衡，但倾向于找兼顾方案

## 常见误用

- **降低标准当作普惠** —— 济弱扶倾不是把所有功能降级到最低水平，而是在保持高水准的同时提供降级方案
- **只关注技术弱势** —— 也要考虑认知弱势（不理解术语）、信息弱势（不了解上下文）
- **把慈善心态代入** —— 这不是"可怜弱者"，而是"系统公平性"。好的系统本就应该覆盖所有使用者

## 组合建议

- 与 **three-principles-analysis**（三民框架）配合：在分析"谁受益"时，特别关注是否遗漏了弱势群体
- 与 **five-power-checks**（五权制衡）配合：将"济弱扶倾"审查加入监察环节
- 与 **power-capability-split**（权能区分）配合：确保弱势使用者也拥有必要的决策权（如退出、撤销、反馈渠道）
