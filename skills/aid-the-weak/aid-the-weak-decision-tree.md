# 济弱扶倾 · 决策树

## 什么时候用？

### ✅ 直接用

- 设计系统时需要考虑可访问性（a11y）
- 方案需要照顾新用户、老年用户、残障用户
- 向下兼容、迁移成本、边缘用户群体的承载
- 技术普惠场景——让技术能力弱的用户也能使用
- 开源项目需要考虑不同技术水平用户的参与门槛

### ⚠️ 配合使用

| 场景 | 配合方法论 |
|------|-----------|
| 弱势方需要被唤起才能参与 | → 配合 **awaken-and-unite** 动员弱势群体 |
| 资源分配时弱势群体被忽视 | → 配合 **three-principles-analysis** 审视权益分配 |
| 帮助弱势群体需要分阶段 | → 配合 **three-phase-governance** 渐进式赋能 |
| 帮助弱势需要对抗强势阻力 | → 配合 **rise-from-defeat** 坚持推进 |

### ❌ 不适用

- **方案本身完全与弱势群体无关**——不需要强行引入这个框架
- **弱势群体需求与其他核心需求严重冲突**——需要权衡，不是一味满足
- **资源极度有限，无法照顾弱势**——优先保证核心功能，济弱是增强而非基础
- **弱势群体自己表示不需要帮助**——尊重对方意愿

---

## 组合使用示例

```
用户说："这个新功能设计得很好，但老用户可能不会用"
  → 济弱扶倾 ✅（审视是否照顾了弱势使用群体）
  → 识别老用户群体的具体障碍（学习成本、操作复杂）
  → 设计降级方案/引导机制/兼容性
  → 配合 three-phase-governance 分阶段推出，降低迁移冲击
```
